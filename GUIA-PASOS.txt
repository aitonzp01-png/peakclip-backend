 1. EJECUTAR SQL EN SUPABASE DASHBOARD (OBLIGATORIO)
─────────────────────────────────────────────────────────────

  Ve a: https://supabase.com/dashboard/project/tjuiourlpbwivjzyewav/sql/new

  Pega y ejecuta este SQL:

  ── INICIO SQL ──
  CREATE TABLE IF NOT EXISTS public.credit_transactions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
    amount INTEGER NOT NULL,
    type TEXT NOT NULL CHECK (type IN ('purchase', 'consume', 'refund')),
    job_id UUID,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  );

  CREATE INDEX IF NOT EXISTS idx_credit_transactions_user_id
    ON public.credit_transactions(user_id);
  CREATE INDEX IF NOT EXISTS idx_credit_transactions_created_at
    ON public.credit_transactions(created_at DESC);

  CREATE OR REPLACE FUNCTION public.handle_new_user()
  RETURNS TRIGGER AS $$
  BEGIN
    INSERT INTO public.users (id, email, credits, plan)
    VALUES (NEW.id, NEW.email, 3, 'free');
    INSERT INTO public.credit_transactions (user_id, amount, type)
    VALUES (NEW.id, 3, 'purchase');
    RETURN NEW;
  END;
  $$ LANGUAGE plpgsql SECURITY DEFINER;
  ── FIN SQL ──

  ✅ Verifica que la tabla se haya creado:

─────────────────────────────────────────────────────────────
 7. SI ALGO FALLA — LOGS DE DEPURACIÓN
─────────────────────────────────────────────────────────────

  Backend logs (Railway):
    1. Ve a: https://railway.app/dashboard
    2. Selecciona peakclip-backend
    3. Ve a "Deployments"
    4. Haz clic en el deployment actual
    5. Ve a "Logs"

  Frontend logs:
    Abre la consola del navegador (F12 > Console)
    Busca errores rojos o mensajes con "ERROR"

  Errores comunes:

  "No credits remaining"
    → Verifica que el usuario tenga créditos en Supabase (Table Editor > users)
    → Si no, inserta manualmente: UPDATE users SET credits = 3 WHERE email = 'tu@email.com'

  "Storage upload failed"
    → Verifica que el bucket 'clips' sea público: GET https://<project>.supabase.co/storage/v1/bucket/clips
    → Debe decir "public": true

  "credit_transactions insert failed (non-fatal)"
    → La tabla no existe → ve al paso 1

  "Network Error" o "Failed to fetch"
    → Verifica CORS: el frontend necesita poder llamar al backend
    → https://peakclip-studio.vercel.app debe estar en ALLOWED_ORIGINS

  "Export error" en FFmpeg
    → Revisa los logs de Railway para ver el error específico de FFmpeg
    → Puede ser codec, formato o ruta de archivo
